# CodeLab101

## 项目现状（给新人的一眼总览）

当前仓库处于**初始化阶段**，核心事实如下：

- 仅包含一个 `README.md` 文件。
- 暂无 `src/`、`docs/`、`tests/`、`scripts/` 等常见目录。
- 暂无可执行代码、依赖清单、构建脚本与自动化测试配置。

这意味着：你接手的是一个“空骨架仓库”，而不是一个已成型的应用项目。

## 目前整体结构

```text
CodeLab101/
└── README.md
```

## 新人需要优先了解的关键点

1. **仓库目标尚未被明确定义**
   - 目前看不到产品定位、技术栈、运行方式等信息。
2. **工程规范尚未落地**
   - 缺少代码风格、分支策略、提交流程、测试门槛等约定。
3. **交付链路尚未建立**
   - 缺少 CI/CD、发布流程、环境配置说明。

## 建议的学习与落地路线（从 0 到 1）

### 第 1 阶段：明确方向（1~2 天）

- 补齐以下最小信息：
  - 这是前端、后端、全栈还是算法项目？
  - 目标用户是谁？
  - 第一版要交付哪些能力？
- 输出一页简版文档：`docs/project-brief.md`（可后续新增）。

### 第 2 阶段：搭建基础工程（1~3 天）

- 初始化目录建议：

```text
CodeLab101/
├── src/
├── tests/
├── docs/
├── scripts/
├── .gitignore
└── README.md
```

- 在 README 中补齐：
  - 本地运行步骤
  - 常用命令
  - 目录说明
  - FAQ

### 第 3 阶段：建立研发闭环（持续迭代）

- 增加基础测试（单元测试优先）。
- 配置 CI（至少做到 push / PR 自动检查）。
- 约定分支模型与提交规范（如 Conventional Commits）。

## 给新人的实操建议

- **先文档，后开发**：先把“做什么、怎么做、做到什么程度”写清楚。
- **先跑通最小闭环**：哪怕功能很小，也要做到“可运行 + 可测试 + 可提交”。
- **每天沉淀**：把踩坑记录到 `docs/`，新同学的学习曲线会明显变短。
